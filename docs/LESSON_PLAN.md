# Lesson Plan: Teach → Model → Guided Practice

Every skill with a problem generator needs a lesson that follows this pattern:

1. **Teach** — Visual explanation of the concept (what it is, why it matters)
2. **Model** — Walk through 2 worked examples step-by-step (show the thinking, not just the answer)
3. **Guided Practice** — Student tries with scaffolding/feedback (already handled by the engine)

---

## Status Key

- **MISSING** — No lesson exists at all. Needs to be written from scratch.
- **WEAK** — Lesson exists but doesn't meet quality bar (missing visuals, only 1 example, example just states answer instead of showing steps, etc.)
- **GOOD** — Lesson meets the Teach/Model pattern.

| `word-problems-add-to-change` | "Add to, change unknown": You know the start and end. Find how much was added. Ex 2: ? + 6 = 13. 13−6=7. |
| `word-problems-take-from-change` | "Take from, change unknown": You know start and end. Find how much was taken. Start − ? = End, so ? = Start − End. | Bar diagram: start → −? → result | Ex 1: There were 15 cookies. Some were eaten. Now there are 9. How many were eaten? 15−9=6. Ex 2: 12 − ? = 5. 12−5=7. |
| `word-problems-take-from-start` | "Take from, start unknown": You know what was taken and what's left. ## Quality Checklist (every lesson must pass)

- [ ] **Teach step** has a visual (dots, equation, shape, clock, number line, bar diagram — NOT just text)
- [ ] **Teach step** explains the WHY, not just the procedure
- [ ] **Teach step** includes a tip for the student
- [ ] **Model step 1** walks through the problem step-by-step (show the thinking, not just the answer)
- [ ] **Model step 2** uses a different problem to show the same strategy generalizes
- [ ] **Model examples** use visuals matching the teach step (if teach used dots, examples should too)
- [ ] **Try-it step** transitions to guided practice with an encouraging prompt
- [ ] No example "gives away" a strategy that bypasses understanding (e.g., "just memorize it")
